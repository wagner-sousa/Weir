📋 DESCRIÇÃO
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Gateway proxy que se posiciona entre agentes de IA e servidores MCP backend.
Agentes conectam-se ao MORPH como um único servidor MCP; o MORPH agrega as
ferramentas de todos os backends configurados e converte cada resultado JSON
em TOON (Token-Oriented Object Notation), reduzindo o uso de tokens em 30-60%.

Backends são alcançados via stdio, HTTP ou SSE (com OAuth opcional).

Repositório: https://github.com/wagner-sousa/morph
Documentação: https://wagner-sousa.github.io/morph/

╔══════════════════════════════════════════════════════════════════════════════╗
║  ARQUITETURA                                                                ║
╚══════════════════════════════════════════════════════════════════════════════╝

Fluxo de requisição:
  AI Agent → MorphMCPServer → Hub → Router → MCPClientRegistry → backend
                                    → ToonConverter (JSON → TOON)

Componentes principais:
  src/index.ts              → CLI entry (start, import), graceful shutdown
  src/hub.ts                → Coordinator central (registry, router, converter,
                              health, metrics, logs, stores, config watcher)
  src/mcp-server/           → Servidor MCP agent-facing
  src/mcp-client/           → Clientes backend (stdio/http/sse + OAuth)
  src/router/               → Mapeamento tool name → (backend, tool)
  src/toon/                 → Conversor JSON→TOON + optimizer + stats
  src/config/               → Schema zod + loader + watcher (hot-reload)
  src/web/                  → API Fastify + WebSocket (:3101) + OAuth routes
  src/persistence/          → SQLite (call history + time-series stats)
  src/logging/              → Logger pino + in-memory log store
  src/examples/             → 5 demo MCP servers (stdio, http, sse, oauth, param)

╔══════════════════════════════════════════════════════════════════════════════╗
║  TRANSPORTES DE BACKEND                                                     ║
╚══════════════════════════════════════════════════════════════════════════════╝

- stdio  → processo filho (command + args)
- HTTP   → requisições REST
- SSE    → Server-Sent Events (streaming)
- OAuth  → Dynamic Client Registration (acoplado a SSE/HTTP)

Resolução de conflitos de tool names:
  1. Aliases explícitos
  2. toolPrefix global
  3. Auto-prefix: ${mcp}_${tool}
  4. allowConflicts: last-wins com warning
